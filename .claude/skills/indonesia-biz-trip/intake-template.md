# 行程信息收集模板

> 此文件定义LLM收集用户信息的规则。LLM读取此文件后，检查用户已提供了哪些项，只追问缺失项，每次一个问题。

## 必填项（9项，全部收集完才进入路线规划）

### 1. 目的地城市/区域
- 示例：雅加达市区、泗水、棉兰、巴厘岛（商务）
- 如果涉及多个城市，请分别说明

### 2. 行程天数 + 出发/返回日期
- 示例：3天，2026-05-10 抵达，2026-05-13 返程
- 用于确定每天的时间窗口

### 3. 航班信息
- 抵达航班：航班号 + 预计到达时间（影响第1天最早可出发时间）
- 返程航班：航班号 + 预计起飞时间（影响最后一天必须到达机场的时间）
- 机场：默认雅加达Soekarno-Hatta，如是其他城市请注明

### 4. 团队人数
- 示例：3人
- 影响包车车型选择和餐饮费用估算

### 5. 交通方式
- 示例：包车（7座MPV，司机熟悉本地路况）
- 可选：打车（Grab/Gojek）、自驾、混合

### 6. 每天必去地点（固定锚点）
- 请按天列出，格式：第N天 - 地点名称（含大致地址或区域）
- 示例：第1天 - Jakarta Convention Center (JCC), Gelora

### 7. 每天候选地点（可去可不去）
- 请按天列出，格式：第N天候选 - 地点名称（区域）
- LLM会根据路线逻辑选择合适的候选点插入行程

### 8. 硬约束
- 示例：
  - 每天单程车程不超过2小时
  - 避开雅加达早高峰（07:00-09:00）和晚高峰（16:30-18:30）
  - 第3天15:30前到达机场
  - 不跨区（如第1天只在Senayan-Sudirman-Kuningan区域）
- 请列出所有强制要求

### 9. 住宿策略
- 每晚住在哪个区域？
- 示例：第1-2晚住雅加达市区Senayan，第3晚换到Pluit（方便第4天早上去机场）
- 或：由LLM根据行程逻辑自动建议

## 可选项（有则读取，没有也可继续生成）

### 用餐偏好
- 正式商务餐 / 快餐优先 / 灵活
- 默认：正式商务午/晚餐

### 特殊注意事项
- 示例：
  - MM2100工厂参观需提前预约
  - SCBD某写字楼停车困难，建议不要开车
  - 某客户只能下午见面

## 追问规则（LLM执行）

1. 逐一检查上方9项必填项，判断用户是否已提供
2. 对于用户已提供的项，不重复询问
3. 每次只问一个缺失项（按1→9顺序）
4. 措辞简洁：直接问，不要长篇解释
5. 9项全部收集完毕后，立即进入路线规划阶段，不再追问
